## 5. Assumptions (Premissas)

### O que é:
Condições consideradas verdadeiras para o desenvolvimento.

- Assume-se que os usuários terão acesso à internet com velocidade mínima para uso de aplicações web.
- Assume-se que os usuários possuem conhecimento básico em informática para operar o sistema sem necessidade de suporte constante.
- Assume-se que cada empresa utilizará o sistema com um único administrador responsável pelo gerenciamento de usuários e permissões.
- Assume-se que os dados de produtos e movimentações inseridos pelos operadores serão fidedignos e inseridos em tempo real.
- Assume-se que o sistema será acessado principalmente via navegador web em dispositivos desktop ou smartphones.
- Assume-se que as pequenas empresas usuárias não possuem integração prévia com ERPs ou sistemas legados complexos.
- Assume-se que o volume de produtos cadastrados por empresa não ultrapassará 10.000 itens ativos no período inicial de uso.
- Assume-se que a equipe de desenvolvimento terá acesso contínuo às ferramentas e ambientes necessários durante todo o projeto.

## 6. Constraints (Restrições)

### O que é:
Limitações técnicas, legais ou de negócio.

### Como fazer:
- Liste tudo que limita o desenvolvimento

### Exemplos:
- O sistema deve ser desenvolvido utilizando tecnologia web.
- O sistema deve rodar em navegadores modernos (Chrome, Firefox).
- O projeto deve ser entregue em até 4 semanas.
- O sistema não poderá utilizar serviços pagos.
